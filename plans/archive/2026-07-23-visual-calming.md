# AnyApp 视觉减压优化实施归档

- 状态：`[done]`（已归档）
- 创建日期：2026-07-23
- 归档日期：2026-07-23
- 项目：`AnyApp`
- 范围：前端视觉减压、按钮内容布局和使用说明信息架构

## 背景与目标

2026-07-22 工作台前端重构以 `[partial]` 归档，功能与基础交互已经完成，但视觉密度未获用户接受。本轮目标是在不改变 Rust、JSON schema、业务逻辑和键盘主流程的前提下，让 AnyApp 更接近安静、清晰的 Windows 桌面启动工具。

需要解决的主要问题：

1. 软件行边框、阴影、渐变和状态徽章重复叠加，形成“盒子墙”。
2. 9–11px 小字过多，控制条、导航和说明区域阅读压力偏高。
3. 主按钮、品牌标和选中状态发光过强，视觉焦点分散。
4. 通用按钮错误地把 SVG 与文字包入同一个 `span`，导致带图标按钮呈现“图标在上、文字在下”。
5. 使用说明把四类内容放在同一滚动正文中，不利于快速定位。

## 实际改动

### 主题与工作台

- `src/styles/anyapp.css`：压平页面、顶栏、导航轨、行悬浮和选中背景，降低蓝色饱和度、边框亮度与阴影重量。
- `src/App.css`：软件卡片改为平静列表行，移除逐行边框、阴影和 hover 位移；保留左侧路径健康轨作为唯一强化识别元素。
- 抬高 meta、说明、导航标签、计数和页脚字号地板。
- 内容控制条由 78px 收紧到 64px，简化窄窗口媒体查询，继续支持 `840 × 600` 最小窗口。

### 信息表达

- `AppRow.tsx`：正常路径不再重复显示健康徽章；便携性从徽章降级为 meta 纯文本，异常状态仍明确显示。
- `LibraryControls.tsx`：移除装饰性 `APPLICATION LIBRARY` kicker，标题和结果计数合并为一行。
- `README.md`：同步“平静列表行”和使用说明 Tab 的当前产品事实。

### 按钮布局修复

- `ui/button.tsx`：主按钮改为扁平色，移除 hover 上浮和过重光晕。
- 根因修复：不再把全部 `children` 包进单个 `span`，让 SVG 与文字成为 `inline-flex` 的同级元素。
- 结果：顶部工具栏、空状态、弹窗和设置中的带图标按钮统一恢复横向“图标 + 文字”；窄侧栏导航保留其独立的纵向布局。

### 使用说明 Tab 化

- 新增 `ui/tabs.tsx`，基于本地 Base UI Tabs 封装统一的 TabList、Tab、Indicator 和 Panel。
- `UsageGuideDialog.tsx` 拆为“添加整理 / 启动操作 / 路径与数据 / 快捷键”四个等宽栏目。
- Tab 图标与文字横排，当前栏目使用低对比蓝色滑块；方向键、焦点和面板关联由 Base UI 统一处理。
- 正文只显示当前主题，弹窗高度和滚动区域保持稳定。

## 用户反馈闭环

1. 用户在真实 Tauri 窗口中指出所有操作按钮的“上图标、下文字”观感异常。
2. 修复通用按钮组件后，运行态截图确认“添加软件”和“扫描目录”等按钮恢复横排。
3. 用户要求使用说明分 Tab；实现后在运行态确认四个栏目可正常显示和切换。
4. 用户随后明确要求归档、总结并推送，本轮按 `[done]` 收口。
