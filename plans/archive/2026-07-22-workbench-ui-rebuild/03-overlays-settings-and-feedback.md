# 阶段 3：弹层、设置与反馈

- 状态：`[done]`
- 前置条件：新 AppShell 和软件列表已稳定

## 目标

用统一的 Dialog、AlertDialog、Sheet、Menu、Field 和 Toast 体系替换当前临时弹层与通知样式，同时保持所有 Tauri command、安全确认和数据恢复能力。

## 弹层分工

| 功能 | 目标组件 | 原因 |
|---|---|---|
| 新增/编辑软件 | Dialog | 单个实体表单，需明确保存/取消 |
| 扫描预览 | 大尺寸 Dialog | 需要候选列表、多选和确认 |
| 分类管理 | Dialog 或 Sheet | 取决于最终编辑密度；不能再嵌入侧栏小按钮后无层级展开 |
| 数据与行为设置 | 右侧 Sheet | 内容长、包含多个分区，适合持续上下文 |
| 删除软件 | AlertDialog | 明确危险确认与焦点管理 |
| 打开外部链接 | AlertDialog | 显示完整 URL 后确认 |
| 行级次级操作 | DropdownMenu | 收敛七个平级按钮 |
| 成功/失败/警告 | Toast/Sonner | 替换自制 fixed notice |

## 软件编辑表单

- 使用 Field、Label、Input、Textarea、Select 和 Switch。
- 字段顺序：名称、路径、分类、描述、标签、网站、收藏。
- 路径选择和文本字段组合使用 FieldAction 或 InputGroup。
- URL 错误必须显示在字段附近，不只显示全局 toast。
- 保存按钮使用与结果一致的动词：`保存软件`、`保存修改`。
- 表单错误发生后焦点移到第一个错误字段。

## 扫描预览

- 目录选择和扫描是两个明确阶段。
- 扫描进行中显示 Spinner/Progress 和“不会执行候选文件”的安全说明。
- 候选列表使用 Checkbox、图标、可编辑名称、路径和分类。
- 选择数量固定显示在 footer，不让确认按钮文案跳动过长。
- 取消或关闭 Dialog 不调用正式保存 command。
- 单项错误或 warning 在候选附近显示；批量 warning 使用可展开区域。

## 分类管理

- 分类列表使用紧凑行，不使用多张嵌套卡片。
- 名称、颜色、软件数量和操作在一行完成。
- 删除被引用分类时，先说明软件将变成未分类，再确认。
- 分类颜色使用受控语义选项，不允许自由输入任意 CSS 色值。

## 设置 Sheet

使用 SettingsBlock 分为：

1. 启动后窗口行为。
2. 紧凑列表模式。
3. 数据目录与配置路径（只读显示）。
4. JSON 导入/导出。
5. 备份与恢复。
6. 批量重新定位。

规则：

- 业务设置与危险数据操作分区。
- 恢复备份和损坏配置重置不得与普通开关并列。
- 绝对路径使用 mono 字体、可复制和截断标题，不占据整页宽度。
- Sheet footer 只放当前上下文需要的操作，不放重复全局按钮。

## 反馈与状态

### Toast

- 启动成功：短提示，可包含 warning。
- 保存成功：说明实际保存对象。
- 扫描/导入完成：显示新增和跳过数量。
- 错误：保留 Rust 返回的明确原因，但不泄露敏感数据。
- 需要用户决策的错误不能只用自动消失 toast，必须保留在当前界面。

### Loading

- 初次加载使用简洁 Skeleton/Spinner，不使用大面积品牌动画。
- 单个软件启动只锁定对应行按钮。
- 保存、扫描、恢复分别有独立 busy state，不能锁死整个应用。

### Recovery

- 损坏配置恢复继续使用独立全屏状态。
- 使用统一 Panel、Button、Badge 和 AlertDialog，但保持 fail-closed 文案。
- 不弱化“普通保存已停止，避免覆盖损坏数据”的提示。

## 可访问性

- Dialog、AlertDialog、Sheet 和 Menu 使用 Base UI 内建焦点管理。
- 验证 Escape、Tab 循环、初始焦点、关闭后焦点恢复。
- 图标按钮必须有中文 `aria-label` 和 Tooltip。
- 状态不能只靠颜色；Badge、StatusDot 和文案必须组合使用。
- destructive 操作不得成为默认聚焦按钮。
- 外部链接确认完整展示目标，不允许 URL 被视觉截断后直接确认。

## 成功标准

- 当前 Radix Dialog/Switch/Tooltip import 清零。
- 全部弹层采用统一 header/body/footer 与宽度规则。
- 行操作不再使用七个独立按钮。
- 设置内容在 `840 × 600` 内可滚动且 footer 始终可达。
- Toast 不替代必须持续显示的错误和确认。
- 导入、恢复、重置和外部链接安全边界不退化。

## 回滚点

- 按弹层逐个迁移，每完成一个弹层立即运行其定向测试。
- 未迁移弹层继续使用旧实现，直到该功能通过 smoke；最终验收前允许短期分阶段共存。
- 只有全部迁移完成后才能移除旧 Radix 依赖和旧 UI 文件。
