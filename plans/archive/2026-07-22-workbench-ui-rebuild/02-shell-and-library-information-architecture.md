# 阶段 2：壳层与软件库信息架构

- 状态：`[done]`
- 前置条件：阶段 1 的组件基础已独立通过 build

## 目标

重构 AnyApp 主窗口，使“搜索 → 判断路径状态 → 启动软件”成为最短、最清晰的操作路径，同时保留全部既有筛选和键盘能力。

## 计划结构

建议把现有 `App.tsx` 拆分为以下职责：

```text
src/
├── App.tsx                              # 查询状态、顶层错误边界和页面编排
├── components/workbench/
│   ├── AnyAppShell.tsx                  # AppShell 组合
│   ├── LibraryToolbar.tsx               # 搜索、扫描、添加、设置
│   ├── FilterRail.tsx                   # 全部/收藏/最近/异常/分类入口
│   ├── LibraryControls.tsx              # 分类、排序、状态汇总
│   ├── PathHealthSummary.tsx            # 紧凑路径健康表达
│   ├── AppList.tsx                      # 列表、空状态、键盘选中
│   ├── AppRow.tsx                       # 高密度软件行
│   └── AppOverflowMenu.tsx              # 次级与危险操作
```

文件名可在实施时按现有命名调整，但职责边界必须成立。

## AppShell

### 顶部工具栏

- 左侧：AnyApp 中性图标、产品名。
- 中部：SearchInput，占据主要可用宽度，保留 `Ctrl+K` / `/` 提示。
- 右侧：扫描、添加软件、设置。
- 顶栏不显示 JSON revision、内部 schema 名称或技术型状态。

### 导航轨

- 全部软件。
- 收藏。
- 最近启动。
- 路径异常。
- 分类入口。

分类数量较多时，不把全部分类永久塞入窄轨；使用分类 Popover/Sheet 或内容区 Select 呈现。

### 内容工具栏

- 当前筛选标题和结果数量。
- 全部/收藏/最近/异常 SegmentedControl；如已在导航轨切换，则避免重复第二套同义导航。
- 分类选择和排序。
- 正常/异常数量使用 StatusDot + 文本，不使用大指标卡。

实施时必须选择一处作为筛选主入口；导航轨与 SegmentedControl 不得重复表达同一个完整状态集合。

## 软件列表

### AppRow 信息层级

第一行：

- 应用图标。
- 名称。
- 收藏状态。
- 分类或路径状态。
- 启动按钮。
- 更多菜单。

第二行：

- 一行描述或解析后的路径。
- 最近启动时间和启动次数作为弱化信息。

### 路径健康轨

- 轨道宽度控制在 3–4px。
- 轨道颜色与 StatusDot tone 来自统一 token。
- 异常项必须同时有文本或图标，不得只依靠颜色。
- 键盘选中使用 focus/selected surface，不覆盖健康语义。

### 操作收敛

始终显示：

- 启动。
- 收藏。
- 更多菜单。

更多菜单依次包含：

1. 在资源管理器中定位。
2. 打开网站（存在时）。
3. 创建桌面快捷方式。
4. 编辑。
5. 分隔线。
6. 删除记录。

删除使用 destructive variant。路径不健康时，启动按钮 disabled，并提供编辑路径或重新定位提示。

## 键盘行为不变量

- `Ctrl+K` / `Cmd+K`：聚焦并全选搜索框。
- `/`：未在输入控件和弹层中时聚焦搜索框。
- `ArrowDown` / `ArrowUp`：移动软件行选中状态。
- `Enter`：启动当前健康软件。
- `Escape`：优先关闭菜单/弹层，其次清除非空搜索或取消当前行选中。
- 选中行必须滚动到可视区域。

迁移到 Menu、Dialog 和 Sheet 后，需要重新确认全局快捷键不会穿透弹层。

## Zustand 与 Query 边界

Zustand 继续只保存：

- 搜索词。
- 筛选与排序。
- 当前键盘选中索引或 ID。
- 临时弹层状态。

TanStack Query 继续保存 Rust/JSON 异步投影。不得为了组件方便把 apps、categories 或 config 再复制进 Zustand。

## 响应式桌面规则

### `1200 × 800`

- 导航轨、顶部工具栏、内容工具栏和列表同时可见。
- 软件行不超过两行文本。
- 更多菜单不挤压名称与路径。

### `840 × 600`

- 导航轨允许压缩为图标模式。
- 顶栏次级按钮可以转为 icon button 或进入 overflow。
- 搜索始终可见。
- 分类与排序可进入 Popover/Sheet，但不能消失。
- 软件列表保持内部滚动，不让窗口整体产生双滚动条。

不再为低于 Tauri 最小窗口的移动端布局牺牲桌面功能；浏览器窄屏只需保证不崩溃和无严重横向溢出。

## 空状态

区分三种情况：

1. 尚无任何软件：解释添加或扫描入口。
2. 搜索/筛选无结果：提供清除条件。
3. 仅路径异常筛选为空：明确说明当前没有需要修复的路径。

空状态文案使用用户可理解的任务语言，不展示 repository、revision 或 schema 术语。

## 成功标准

- 主窗口完成新 AppShell 切换。
- 软件行最多显示三个操作入口。
- 搜索、筛选、排序和分类在 `840 × 600` 可达。
- 键盘启动流程保持完整。
- 当前全部、收藏、最近、失效和分类筛选测试继续通过。
- 视觉中不存在重复大卡片和无业务意义的指标面板。

## 回滚点

- 在新壳层通过 build 和基础交互 smoke 前，旧 `App.tsx` / `App.css` 保持可切回。
- 入口切换单独提交为一个可逆步骤；本计划本身不授权 commit。
- 回滚只切换前端入口，不触碰用户 JSON 或 Rust repository。
